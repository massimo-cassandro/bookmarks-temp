Squash and Stretch

About this lesson (info)

In addition to the technical implementation side, it’s also very important that we learn the design side. Very few of us have the luxury of working with a professional motion designer, after all!

We cover design principles throughout the course, and there’s also a bonus feature that focuses on animation design in more depth. This is the 8th lesson (out of 19 total) from that bonus feature.

Have you ever heard of Disney’s 12 Basic Principles of Animation (opens in new tab)?

It’s a collection of animation best practices created in 1981 by two Disney animators, intended to be used by the folks who produce animated cartoon movies like Aladdin or Beauty And The Beast.

Most of the rules aren’t directly applicable to us as web developers, and others are things we’ve already covered (like easing curves!). But the very first rule, “Squash and Stretch”, is super worthwhile.

Here’s a demo showing how this rule can be implemented into a bouncing ball animation:

0%
500ms

As you increase the “Squash/Stretch Amount”, the ball starts to flatten like a water balloon when it hits the ground, growing horizontally and shrinking vertically. As it bounces back up, it elongates, becoming long and skinny.

At the high end, the squash/stretch is exaggerated to the point of being ridiculous 😅. I chose extreme values to make sure the effect is clear, but in most realistic scenarios, I’d probably choose something in the 25-50% range. This effect often works best when it’s pretty subtle.

Also, as we saw earlier when we tried to animate bouncing balls, it’s surprisingly difficult to convincingly simulate gravity and physics using Bézier curves. I think this animation looks good at fast speeds, but it starts to feel a lot less convincing with slower durations. Fortunately, most of the actual use cases I have for this squashing/stretching technique don’t involve gravity or bouncing at all.

How does that bouncing ball work?? (info)

The rest of this lesson will focus on the practical real-world ways I use this technique in my own work, but if you’re curious how this “bouncing ball” demo works, I’ll include annotated source code here:

Code Playground

<style>
/*
The basic setup here is that we have a “bouncer”
that moves up and down, consistently. Within that
element, we have a “ball“ that does the squashing
and stretching.

It might theoretically be possible to combine them
in a single element/animation, but it would make
the code far more complicated.
*/
.bouncer {
--curve: cubic-bezier(0.123, 0.5, 0.5, 1);

width: 50px;
height: 50px;
will-change: transform;
/*
Like we saw in the Bouncing Balls exercise, this
keyframe animation moves the element up and down,
using an “ease-out” style easing curve to simulate
gravity.
*/
animation:
bounce 400ms var(--curve) infinite alternate;
}

.ball {
/*
Change this value to affect the magnitude
of the bounce:
*/
--amount: 0.5;

width: 100%;
height: 100%;
background: hsl(45deg 100% 50%);
border-radius: 50%;
animation: squash 800ms linear infinite;
transform-origin: center bottom;
}

/*
This keyframe causes an item to move up and down.
It stops moving for a brief moment (2% of the
total duration), so that the “squash” appears to
happen on the ground, rather than in the air.
*/
@keyframes bounce {
0% {
transform: translateY(0);
}
2% {
transform: translateY(0);
}
100% {
transform: translateY(-100px);
}
}

/*
This keyframe is very scary. I apologize in
advance 😅.

Here’s the basic idea: when the animation starts,
this ball is fully squashed, like a water balloon
hitting a window. Over the first 10%, it quickly
transitions to being long and skinny. Finally, the
bulk of the animation duration is spent returning
to its natural shape, before the whole thing
happens again on the next iteration.

For this particular effect, I needed the squash
amount to be dynamic, for the “Squash/Stretch
Amount” slider. That slider controls the --amount
variable, and I’m using “calc” to interpolate that
value into squash/stretch amounts. You could
simplify this code by picking static values instead.
*/
@keyframes squash {
0% {
transform: scale(
calc(1 + var(--amount) / 2),
calc(1 - var(--amount) / 2)
);
}
10% {
transform: scale(
calc(-0.5 * var(--amount) + 1),
calc(0.5 * var(--amount) + 1)
);
}
100% {
transform: scale(1, 1);
}
}
</style>

<div class="wrapper">
<div class="bouncer">
<div class="ball"></div>
</div>
</div>

This code is quite complicated, and honestly, I don’t think you really need to understand it. Like I mentioned above, I don’t actually use this technique for bouncing balls! The rest of this lesson contains much-more-practical use cases for this technique, and they’re easier to understand than this.

But if you’d like to understand it, I’d suggest temporarily removing the animation: bounce declaration. That way, you can see the squash animation on its own.

Implementing stretchy arrows

Let’s look at how we can create effects like this!

First, we need an icon to work with. My favourite icon pack these days is Lucide (opens in new tab), a fork of the legendary Feather Icons pack, which adds >1000 new icons in the same lovely style.

So, let’s grab the arrow-right icon (opens in new tab) from Lucide, downloading it as an SVG. After reformatting the code and removing unnecessary attributes like xmlns, here’s what we’re left with:

<svg
  width="24"
  height="24"
  viewBox="0 0 24 24"
>
  <path
    d="
      M 5,12
      h 14
    "
  />
  <path
    d="
      M 12,5
      l 7,7
      l -7,7
    "
  />
</svg>

That first <path> is the main shaft. The second <path> is the arrow tip. This code makes use of SVG syntactic sugar features; for example, that h command draws a horizontal line that is 14px long, relative to the starting position of (5, 12).

Before we get to a full production-ready implementation, let’s start by keeping it as simple as possible, so that we can focus on the core concept.

Here’s the approach:

Code Playground

<style>
html {
--spring-easing: linear(0, 0.009, 0.036 1.6%, 0.148 3.5%, 0.757 10.1%, 0.979 13%, 1.06, 1.121, 1.163, 1.187 18.9%, 1.194, 1.195, 1.19, 1.181 22.8%, 1.151 24.9%, 1.054 30.2%, 1.011 32.9%, 0.979 35.9%, 0.97, 0.964 39.1%, 0.962, 0.965 43.2%, 0.997 53.1%, 1.007 59.1% 63.6%, 0.999 78.8%, 1);
--spring-duration: 1s;
}
.shaft, .tip {
transition: d var(--spring-easing) var(--spring-duration);
}
.btn:hover .shaft {
d: path("\
M 5,12\
h 17\
");
}
.btn:hover .tip {
d: path("\
M 15,7\
l 7,5\
l -7,5\
");
}
</style>

This version doesn’t include the fancy “Boop” technique I use on my blog. It also doesn’t work in Safari, since Safari doesn’t yet support CSS transitions on path instructions (at least, not as I write this in late 2025).

The basic premise here is that we’re overwriting the path definitions for both <path> shapes when the button is hovered. Taking the shaft as an example, we still start at the same point, (5,12), except now we draw a 17px-long line, 3px longer.

It took me a while to get truly comfortable with path syntax, and even now, it was non-trivial for me to come up with those overrides. I had to do a fair amount of trial and error, to find the new set of values that drew the squished arrow!

Like I mentioned earlier, this version isn’t quite production-ready. At the very least, we’d want to consider motion preferences. The fact that this effect doesn’t work in Safari may or may not be a dealbreaker; for purely-cosmetic effects like this, we don’t necessarily need perfect browser support. But in my own work, I generally do try to support all modern browsers.

Here’s a more sophisticated version that includes the “Boop” effect, using Motion for universal browser support:

Code Playground

import { animate } from 'motion';
import './reset.css';
import './styles.css';

const btn = document.querySelector('.btn');
const shaft = btn.querySelector('.shaft');
const tip = btn.querySelector('.tip');

const SPRING_CONFIG = {
type: 'spring',
stiffness: 300,
damping: 16,
};

btn.addEventListener('mouseenter', (event) => {
const prefersReducedMotion = checkPrefersReducedMotion();
if (prefersReducedMotion) {
return;
}

animate(shaft, { d: 'M 5,12 h 17' }, SPRING_CONFIG);
animate(tip, { d: 'M 15,7 l 7,5 l -7,5' }, SPRING_CONFIG);

window.setTimeout(() => {
animate(shaft, { d: 'M 5,12 h 14' }, SPRING_CONFIG);
animate(tip, { d: 'M 12,5 l 7,7 l -7,7' }, SPRING_CONFIG);
}, 150);
});

function checkPrefersReducedMotion() {
return !window.matchMedia(
'(prefers-reduced-motion: no-preference)'
).matches;
}
